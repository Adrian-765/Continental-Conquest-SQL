💾 Consultas SQL
norte_n7.sql
1 · Peso por posición — GROUP BY
SELECT posicion,
       SUM(peso) total
FROM cargas_mar_norte
GROUP BY posicion;
2 · Diferencia babor/estribor — ABS
SELECT ABS(
  (SELECT SUM(peso) FROM cargas_mar_norte
   WHERE posicion = 'Babor')
  -
  (SELECT SUM(peso) FROM cargas_mar_norte
   WHERE posicion = 'Estribor')
) AS diferencia;
3 · Detectar situación crítica
SELECT 1
FROM cargas_mar_norte
GROUP BY 1
HAVING ABS(
  SUM(CASE WHEN posicion='Babor'
       THEN peso ELSE 0 END)
  -
  SUM(CASE WHEN posicion='Estribor'
       THEN peso ELSE 0 END)
) > 10;
4 · Diferencia total — CASE WHEN
SELECT ABS(
  SUM(CASE WHEN posicion='Babor'
       THEN peso ELSE 0 END)
  -
  SUM(CASE WHEN posicion='Estribor'
       THEN peso ELSE 0 END)
) AS diferencia_total
FROM cargas_mar_norte;
📋 Estructura de tablas
CREATE TABLE cargas_mar_norte (
    id_carga INT PRIMARY KEY,
    peso     INT,
    posicion VARCHAR(20)
    -- 'Babor' | 'Centro' | 'Estribor'
);
🔍 Análisis Táctico
GROUP BY posicion genera tres grupos: Babor (40+30=70t), Centro (20t), Estribor (35t). SUM(peso) suma las cargas de cada posición. Esta es la base para calcular el desequilibrio.
ABS() devuelve siempre un valor positivo independientemente de qué lado pese más. Subconsulta babor: 70t. Subconsulta estribor: 35t. Resultado: ABS(70-35) = 35 toneladas de desequilibrio.
GROUP BY 1 agrupa toda la tabla en un único grupo. HAVING aplica el CASE WHEN completo y comprueba si la diferencia > 10. Si devuelve una fila → situación crítica confirmada. Si está vacía → el barco está equilibrado.
CASE WHEN posicion='Babor' THEN peso ELSE 0 END suma solo los pesos de babor (el ELSE 0 ignora las otras posiciones). Lo mismo para estribor. ABS de la resta da el desequilibrio total en una sola consulta sin subconsultas.
