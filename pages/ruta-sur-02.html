1915–1916 — Novotroitskaya · Rostov · Don · Hacia Moscú

El puerto de Novorossiysk queda atrás, y la milicia avanza con paso medido hacia el corazón del Imperio Ruso. Las primeras carreteras y ferrocarriles revelan un paisaje vasto y ajeno: llanuras interminables, bosques que bordean el Don, pequeños pueblos bajo la atenta mirada del Zar.

Al llegar a Novotroitskaya establecemos campamento. Los contactos bolcheviques manejan información dispersa: soldados, armas, suministros, posiciones estratégicas en rutas ferroviarias y fluviales. "Organizar el caos antes de que despierte el Zar."

La campaña avanza. Las ciudades clave: Rostov (control del Don), el Volga (arteria fluvial), Novocherkassk (centro militar) y el objetivo final: Moscú, el corazón del poder zarista.

La ofensiva hacia Moscú exige precisión total. Cada error puede costar la misión.

A · Fuerza por tipo
SELECT tipo,
       SUM(soldados) AS total_soldados,
       SUM(municion)  AS total_municion
FROM unidades
GROUP BY tipo;
B · Asignados a una ruta
SELECT SUM(soldados_asignados) AS soldados_totales,
       SUM(municion_asignada)  AS municion_total
FROM asignaciones
WHERE id_ruta = 4; -- Moscú
C · Solo rutas críticas
SELECT *
FROM rutas
WHERE prioridad = 'Crítica';
D · Procedimiento de validación
CREATE PROCEDURE validar_asignacion(
    IN unidad_id     INT,
    IN soldados_nuevos INT
)
BEGIN
  DECLARE capacidad    INT;
  DECLARE ya_asignados INT;
  SELECT soldados INTO capacidad
  FROM unidades WHERE id_unidad = unidad_id;
  SELECT COALESCE(SUM(soldados_asignados),0)
  INTO ya_asignados
  FROM asignaciones WHERE id_unidad = unidad_id;
  IF ya_asignados + soldados_nuevos > capacidad THEN
    SIGNAL SQLSTATE '45000'
    SET MESSAGE_TEXT =
      'Asignación excede capacidad';
  END IF;
END;

🔍 Análisis Clasificado
GROUP BY tipo agrupa: extranjera (80+40=120 soldados), logística (25), local (100+60+80=240). Permite ver qué tipo de unidad tiene más fuerza antes de asignarlas a Moscú.
WHERE id_ruta = 4 filtra solo las asignaciones hacia Moscú. SUM suma los soldados y munición asignados en esa ruta. Si la tabla asignaciones está vacía, el resultado será NULL o 0.
Solo la ruta 4 (Moscú, 800 km) tiene prioridad 'Crítica'. Este SELECT devolverá únicamente esa fila. Es la ruta que concentra todos los recursos finales.
El procedimiento usa DECLARE para variables locales, SELECT INTO para capturar valores y una estructura IF con SIGNAL para lanzar un error si la asignación supera la capacidad real de la unidad.
