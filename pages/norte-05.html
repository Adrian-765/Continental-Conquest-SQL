He elegido el norte.

El Mediterráneo ofrecía una ruta más cómoda, pero la comodidad en guerra es una trampa. Aquí, en Dinamarca, todo parece en calma. Las calles están iluminadas. Los puertos funcionan. Nadie dispara.

Y, sin embargo, todo está vigilado.

La neutralidad danesa no es una protección; es un equilibrio frágil entre potencias que observan cada movimiento. Los estrechos hacia el Báltico son arterias estratégicas. Quien entra o sale hacia Rusia deja rastro.

Llegamos a Copenhague bajo identidades prestadas. Sellos alemanes. Documentación precisa. Nada puede parecer improvisado.

Seamus organizó la reunión con un comerciante naval. No comparte nuestra causa. Comparte nuestro interés. Rusia debe debilitarse. Ese es el único punto en común.

Extendió sobre la mesa registros de rutas, barcos, permisos y cargas. No quería discursos. Quería coherencia.

—"Si vuestros documentos no se sostienen entre sí, no cruzaréis el Báltico."

Comprendí entonces que la verdadera prueba no era política. Era logística. Cada barco debía justificar su peso. Cada carga debía coincidir con su permiso. Cada autorización debía resistir una inspección británica.

Mañana demostraré que entiendo esa red invisible. Si fallo, no habrá segunda oportunidad.

💾 Consultas SQL
norte_n5.sql
1 · Ocupación real por barco
SELECT nombre_barco,
       SUM(peso) total,
       ROUND(SUM(peso)/capacidad_maxima*100,2)
         AS ocupacion
FROM barcos
JOIN cargas USING(id_barco)
GROUP BY id_barco;
2 · Detectar sobrepeso — HAVING
SELECT nombre_barco
FROM barcos
JOIN cargas USING(id_barco)
GROUP BY id_barco
HAVING SUM(peso) > capacidad_maxima;
3 · Sin carga — LEFT JOIN + HAVING
SELECT nombre_barco
FROM barcos
LEFT JOIN cargas USING(id_barco)
GROUP BY id_barco
HAVING COUNT(id_carga) = 0;
4 · Puede zarpar legalmente
SELECT nombre_barco
FROM barcos
JOIN permisos USING(id_barco)
JOIN cargas   USING(id_barco)
WHERE estado_permiso = 'Aprobado'
GROUP BY id_barco
HAVING SUM(peso) <= capacidad_maxima;
📋 Estructura de tablas
CREATE TABLE barcos (
    id_barco         INT PRIMARY KEY,
    nombre_barco     VARCHAR(50),
    bandera          VARCHAR(30),
    capacidad_maxima INT
);
CREATE TABLE cargas (
    id_carga  INT PRIMARY KEY,
    id_barco  INT,
    peso      INT
);
CREATE TABLE permisos (
    id_permiso    INT PRIMARY KEY,
    id_barco      INT,
    estado_permiso VARCHAR(20)
);
🔍 Análisis Táctico
WHERE filtra filas antes de agrupar y no puede usar SUM(). HAVING filtra después de GROUP BY y sí puede comparar agregaciones como SUM(peso) con capacidad_maxima. Es el concepto central de este nivel.
Skagen Star tiene SUM(peso)=120+90=210 pero capacidad_maxima=200. La consulta HAVING SUM(peso) > capacidad_maxima devuelve solo el Skagen Star como barco sobrecargado.
LEFT JOIN incluye todos los barcos aunque no tengan cargas. COUNT(id_carga) = 0 en HAVING detecta los que no tienen ninguna carga asignada. Si todos tienen carga, el resultado estará vacío.
Aurora Borealis: permiso aprobado + SUM(peso)=200 ≤ 300. Es el único que cumple ambas condiciones. Skagen Star tiene permiso pero sobrepeso. Baltic Wind tiene permiso pendiente.
