💾 Consultas SQL
norte_n8.sql
1 · Peso declarado vs real — JOIN
SELECT d.id_doc
FROM documentos_transito d
JOIN carga_real c USING(tipo_carga)
WHERE d.peso <> c.peso_total;
2 · Secuencia de tránsito — ORDER BY
SELECT *
FROM documentos_transito
ORDER BY id_doc;
3 · Detectar salto incoherente
SELECT *
FROM documentos_transito
WHERE origen = 'Dinamarca'
  AND destino = 'Suecia';
4 · Total transportado — SUM
SELECT SUM(peso) AS total_transito
FROM documentos_transito;
📋 Estructura de tablas
CREATE TABLE documentos_transito (
    id_doc     INT PRIMARY KEY,
    origen     VARCHAR(50),
    destino    VARCHAR(50),
    tipo_carga VARCHAR(30),
    peso       INT
);
CREATE TABLE carga_real (
    tipo_carga VARCHAR(30),
    peso_total INT
);
🔍 Análisis Táctico
JOIN une documentos_transito con carga_real por tipo_carga. WHERE d.peso <> c.peso_total busca discrepancias. En este caso ambos documentos declaran 125 y la carga real es 125, así que el resultado está vacío — no hay fraude documental.
ORDER BY id_doc muestra los documentos en orden: Dinamarca→Noruega (id=1) y luego Noruega→Suecia (id=2). La secuencia es coherente: el destino de cada documento es el origen del siguiente.
WHERE origen='Dinamarca' AND destino='Suecia' busca un salto que omitiera Noruega. En los datos actuales no existe ese documento, así que el resultado está vacío — no hay salto incoherente. Si existiera, las autoridades detectarían el fraude.
SUM(peso) suma 125+125=250 unidades totales. Este número representa el tránsito acumulado a lo largo de toda la ruta escandinava desde Dinamarca hasta Suecia.
